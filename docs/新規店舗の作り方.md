# Review Booster：新規店舗の作り方（開発テンプレート）

**目的**: 他店舗を追加するときの「決めること一覧」「AIとの協議フロー」「実装の当て先」を一箇所にまとめ、無駄なく同じ流れで作れるようにする。

---

## 1. テンプレートの定義

- **マスターテンプレ**: `src/config/stores.ts` の **`_template`** および [stores.template.example.md](stores.template.example.md) のコピー用ブロック。
- **UI**: `src/app/[storeId]/page.tsx` 共通。質問の並びは固定（評価 → ご本人は？ → 来店タイプ → 良かったポイント → 誰と → 推しスタッフ）。項目の追加・削除はせず、**ON/OFF と文言の差し替えのみ**。
- 詳細な方針は [テンプレート方針.md](テンプレート方針.md) を参照。

---

## 2. 決めておく必要があること一覧

新規店舗を作る前に、**フォームのデザインから口コミ出力まで**で決める項目を下表にまとめる。Gemini に「ユーザーと協議してこれらを決め、最終仕様を出力させる」ために使う。

### A. フォーム（表）のデザイン

| # | 項目 | 内容 | 実装の場所 |
|---|------|------|------------|
| 1 | 表示する質問の有無 | 評価 / 良かったポイント / 誰と / ご本人は？ / 来店タイプ / 推しスタッフ の ON/OFF | `stores.ts` の `features.*.enabled` |
| 2 | 良かったポイントの選択肢 | フォームに並べる選択肢（日本語・英語）。4つ想定。 | `features.keywords.options.ja` / `.en` |
| 3 | 誰との選択肢 | 例: 友達 / 同僚 / 恋人 / 一人。使う場合のみ文言を決定。 | `features.companion.options` |
| 4 | ご本人は？（性別）の選択肢 | 例: 男性 / 女性。 | `features.gender.options` |
| 5 | 来店タイプの選択肢 | 例: 地元・沖縄住み / 旅行・観光。店舗に合う聞き方に。 | `features.visitType.options` |
| 6 | 推しスタッフの有無と placeholder | 入力欄を出すか。出す場合のプレースホルダー（ja/en）。 | `features.staffName.enabled` / `placeholder` |
| 7 | レイアウト | 1ページ構成か、複数ステップか。 | `singlePageLayout: true/false` |

### B. 口コミ生成（出力）で使うもの

| # | 項目 | 内容 | 実装の場所 |
|---|------|------|------------|
| 8 | 店舗説明（プロンプト用） | 口コミ生成時にAIに渡す店舗の説明（場所・コンセプト・客層・価格など）。日本語・英語。 | `promptContext.ja` / `promptContext.en` |
| 9 | キーワードの言い換え | 各「良かったポイント」選択肢に対応する、口コミ本文で使う表現のバリエーション（各2〜5個）。 | `actions.ts` の `keywordVariations`（店舗分岐で追加） |
| 10 | 店舗固有のプロンプト指示 | 口コミのトーン・場所の言い方・入れたい要素・禁止事項・絵文字の目安。 | `actions.ts` の `storeSpecificSettings` または storeId 分岐内の指示文 |
| 11 | 誰と / 性別 / 来店タイプの扱い | 選択肢をプロンプトや英語生成でどう使うか（例: 友達→friends）。 | `actions.ts` 内の companionEn やプロンプトの `${companion}` 等 |
| 12 | 英語口コミ用キーワード説明 | 英語生成時、各キーワードをどう強調するかの英文。 | `actions.ts` の `keywordContextsEn`（店舗ごとに追加する場合） |

### C. その他（識別・運用）

| # | 項目 | 内容 | 実装の場所 |
|---|------|------|------------|
| 13 | 店舗ID | 英小文字・ハイフン。URL と `storeId` 分岐に使用。 | `stores.ts` のキー、`getStoreConfig(storeId)` |
| 14 | 店舗名・ロゴ・テーマ色 | 表示名（ja/en）、ロゴ画像パス、primaryColor / secondaryColor / logoGlow。 | `stores.ts` の `name` / `nameEn` / `logoPath` / `theme` |
| 15 | Googleマップ | 投稿導線用のURLと placeId。 | `googleMapsUrl` / `placeId` |

## 3. 新規店舗を作る時の流れ（3ステップ）

### ステップ1: 設計項目の確認

- 上記 **「2. 決めておく必要があること一覧」** の A〜C を確認する。
- 質問を何にするか・選択肢の文言・口コミに載せるキーワードは**セットで決める**（フォームの選択肢＝口コミの種になるため）。

### ステップ2: Gemini でユーザーと協議し、最終仕様を出力させる

- 下記 **「4. Gemini に渡す指示文」** をコピーし、必要なら店舗名や前提を書き換える。
- Gemini に貼り付け、ユーザー（依頼者）と対話しながら上記の項目をすべて決めさせる。
- 最後に **「最終仕様」を1本のドキュメントとして出力**させる（開発がそのまま実装に使える形式で）。

### ステップ3: 実装に反映する

- **フォーム・選択肢・placeholder・店舗説明・テーマ・レイアウト**  
  → `src/config/stores.ts` の `stores["店舗ID"]` に、[stores.template.example.md](stores.template.example.md) をベースに追記・編集。
- **キーワード言い換え・口コミトーン・店舗分岐**  
  → `src/app/actions.ts` の `keywordVariations` および `storeId` に応じた分岐（`storeSpecificSettings` やプロンプト内の店舗別指示）を追加。
- **ロゴ**  
  → `public/{店舗ID}-logo.png` を配置し、`logoPath` に指定。
- 動作確認は `http://localhost:3000/{店舗ID}` で実施。

## 4. Gemini に渡す指示文（テンプレート）

以下をそのまま Gemini に渡すか、店舗名などだけ差し替えて使う。

```
【役割】
あなたは、Review Booster の「新規店舗」の質問内容・選択肢・口コミ生成の設計を、ユーザーと協議して決め、最終仕様として出力する担当です。

【前提】
- フォームは次の項目の ON/OFF と文言を決める：評価（星）、良かったポイント（複数選択）、誰と、ご本人は？（性別）、来店タイプ、推しスタッフ（任意入力）。
- 口コミ生成では、選択された「良かったポイント」が言い換えバリエーションとして文中に反映されます。フォームの選択肢と「口コミに載せたいキーワード・言い回し」をセットで決める必要があります。

【あなたがユーザーと協議の上、決めて最終出力する項目】
1. 表示する質問項目の ON/OFF（評価 / 良かったポイント / 誰と / ご本人は？ / 来店タイプ / 推しスタッフ）
2. 「良かったポイント」の選択肢（日本語・英語）と、口コミで使う言い換えバリエーション（各選択肢につき2〜5個）
3. 「誰と」の選択肢（使う場合）。口コミでの言及の仕方（例: 友達→「友達と」）
4. 「ご本人は？」「来店タイプ」の選択肢（使う場合）
5. 「推しスタッフ」を出す場合の入力欄のプレースホルダー（日本語・英語）
6. 口コミ生成用の店舗説明（promptContext）：日本語・英語。店の場所・コンセプト・客層・価格・雰囲気を簡潔に
7. 口コミのトーン・入れたい要素・避けたい表現・絵文字の目安（プロンプト指示として書く短い仕様）
8. 店舗ID・店舗名（表に出る名前）・レイアウト（1ページ or 複数ステップ）

【進め方】
1. 上記の項目を順に、ユーザーに質問または提案して協議する。
2. ユーザーの要望を反映して案を更新する。
3. すべての項目が決まったら、「最終仕様」を次の形式で1つにまとめて出力する。

【最終出力の形式】
- 見出し「【新規店舗名】Review Booster 最終仕様」で始める。
- 以下のセクションを必ず含める（マークダウンでよい）：
  - フォーム: 表示する質問一覧、各選択肢の文言（ja/en）、推しスタッフの placeholder（ja/en）
  - 口コミ用データ: 良かったポイントの「選択肢 → 口コミ言い換えバリエーション」の一覧、誰と/性別/来店タイプの選択肢とプロンプトでの扱い
  - 店舗説明（promptContext）: 日本語・英語の文
  - 口コミトーン・ルール: プロンプトに渡す短い指示（箇条書きでOK）
  - その他: 店舗ID、店舗名、レイアウト（1ページ/複数ステップ）
- 開発が stores.ts と actions.ts にそのまま反映できるよう、選択肢やキーワードは具体的な文字列で書く。
```

## 5. 実装時の参照（どこに何を書くか）

| 決めた内容 | 書く場所 |
|------------|----------|
| 店舗ID・名前・ロゴ・Googleマップ・テーマ | `src/config/stores.ts` の `stores["店舗ID"]` |
| 各質問の ON/OFF | `stores.ts` の `features.rating/keywords/companion/gender/visitType/staffName.enabled` |
| 良かったポイントの選択肢（ja/en） | `features.keywords.options.ja` / `.en` |
| 誰と / 性別 / 来店タイプの選択肢 | `features.companion/gender/visitType.options` |
| 推しスタッフの placeholder | `features.staffName.placeholder.ja` / `.en` |
| 店舗説明（口コミ用） | `promptContext.ja` / `.en` |
| 1ページ構成にするか | `singlePageLayout: true` または省略（false） |
| キーワード言い換えバリエーション | `src/app/actions.ts` の `keywordVariations` に店舗用キーを追加 |
| 口コミのトーン・場所・禁止事項など | `actions.ts` 内の storeId 分岐（`storeSpecificSettings` やプロンプト文字列） |
| 英語用キーワード説明 | `actions.ts` の `keywordContextsEn`（英語生成を行う場合） |
