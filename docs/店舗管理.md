# Review Booster 配下の店舗一覧と追加ルール

**Review Booster**（Map Engine のサービス）の下に、**店舗**が並ぶ構成です。  
各店舗は同じアプリ内で店舗IDごとに切り替えています。

- **テンプレート**: BARVEL用の Review Booster をテンプレートとする。新規店舗は**マスターテンプレをコピーしたうえで**変更を加える。アンケート項目は固定。マスターテンプレは `stores.ts` の **`_template`** および [docs/stores.template.example.md](stores.template.example.md)。**他店舗を作る時の流れ（決めること一覧・Gemini 協議・実装）**は [docs/新規店舗の作り方.md](新規店舗の作り方.md)。詳細は [docs/テンプレート方針.md](テンプレート方針.md) を参照。

---

## 構成

```
Map Engine（事業）
└── Review Booster（サービス）
    ├── /                  … LP（サービス紹介・作成中）
    ├── BARVEL KOZA        … /barvel-koza
    ├── BAR REPLICA        … /bar-replica
    ├── CEBUOCTO           … /cebuocto
    └── （今後の店舗）     … /{店舗ID}
```

- **ルート `/`**: Review Booster の LP（サービス紹介）。現在は作成中プレースホルダー。
- **店舗**: すべて **`/{店舗ID}`**。例: `https://（ドメイン）/barvel-koza`、`/bar-replica`、`/cebuocto`。
- 新規店舗は **Review Booster の下** に追加し、パスは **`/{店舗ID}`** にします。

## 新規店舗を追加するとき（Review Booster の下に追加）

1. **`src/config/stores.ts` に追加**
   - `stores` に新しいキー（店舗ID、英小文字・ハイフン）を追加。
   - `StoreConfig` に従い、`id`, `name`, `nameEn`, `logoPath`, `googleMapsUrl`, `theme`, `features`, `promptContext` を書く。
   - 1ページ構成にしたい場合は `singlePageLayout: true` を指定。

2. **ルーティング**
   - **共通UIでよい場合**: 追加するだけで **`/[店舗ID]`** で表示される（例: `/new-store`）。  
     ルート `/` は LP のため、全店舗は必ず `/{店舗ID}` の形式。
   - **店舗ごと専用デザインにしたい場合**: `src/app/[店舗ID]/page.tsx` は既に動的ルートなので、`stores.ts` に設定を足せば表示される。  
     デザインを完全に分けたい場合は、CEBUOCTO と同様に **`src/app/[店舗ID]/page.tsx`** の分岐でレイアウトを切り替えるか、**`src/app/（店舗ID）/page.tsx`** を新規作成する。

3. **ロゴ**
   - `public/{店舗ID}-logo.png` を置き、`stores.ts` の `logoPath` に `"/{店舗ID}-logo.png"` を指定。

4. **口コミ生成**
   - `generateReview(..., storeId)` に新しい店舗IDを渡すだけでよい。  
     `actions.ts` で `storeId` に応じた文言が出るので、必要なら `actions.ts` 内の店舗分岐や `keywordVariations` に追加する。

- **CEBUOCTO**: 上記のとおり `/cebuocto` で提供。専用ページ・南国テーマ・正しい Google マップリンク済み。
- **今後の店舗**: 上記「新規店舗を追加するとき」に従い、**Review Booster の下に `/{店舗ID}` で追加**する。

URLはすべて **（本番ドメイン）/{店舗ID}**。ルート `/` は LP 用です。ドメイン・URLの変更手順は [URL・ドメインの変更.md](URL・ドメインの変更.md) を参照。
